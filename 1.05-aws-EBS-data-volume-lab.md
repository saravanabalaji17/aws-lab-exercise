# AWS Hands-on Lab

## EC2 + EBS Volume + Filesystem + Mount + Reverse Cleanup

### Lab Naming Convention

Use your name as prefix.

Example:

| Resource     | Name                          |
| ------------ | ----------------------------- |
| EC2 Instance | `studentname-ec2`             |
| Root Volume  | `studentname-ec2-root-volume` |
| Data Volume  | `studentname-ec2-data-volume` |

---

# Part 1 — Create EC2 Instance

Login to **Amazon Web Services Console**

Go to **Amazon EC2**

### Steps

1. Click **Launch Instance**

2. Configure:

| Setting        | Value                  |
| -------------- | ---------------------- |
| Name           | `studentname-ec2`      |
| AMI            | Amazon Linux / Ubuntu  |
| Instance type  | t2.micro               |
| Key pair       | create or use existing |
| Network        | default VPC            |
| Security group | allow SSH (22)         |

3. Storage

Rename root volume:

```
studentname-ec2-root-volume
```

4. Launch instance

---

# Part 2 — Create Data EBS Volume

Go to **EC2 → Volumes**

Click **Create Volume**

| Setting           | Value                         |
| ----------------- | ----------------------------- |
| Name              | `studentname-ec2-data-volume` |
| Size              | 5 GiB                         |
| Type              | gp3                           |
| Availability Zone | Same as EC2                   |

Create volume.

---

# Part 3 — Attach Volume to EC2

Select volume → **Actions → Attach Volume**

| Setting     | Value           |
| ----------- | --------------- |
| Instance    | studentname-ec2 |
| Device name | `/dev/sdf`      |

Attach.

---

# Part 4 — Connect to EC2

SSH into instance

```bash
ssh -i key.pem ec2-user@public-ip
```

Check disks

```bash
lsblk
```

Example output

```
xvda   8:0   0  10G  0 disk
└─xvda1

xvdf   8:80  0   5G  0 disk
```

`xvdf` = new EBS disk

---

# Part 5 — Create Filesystem

Create filesystem

```bash
sudo mkfs -t ext4 /dev/xvdf
```

Verify

```bash
lsblk -f
```

---

# Part 6 — Create Mount Directory

```bash
sudo mkdir /data
```

---

# Part 7 — Mount Volume

```bash
sudo mount /dev/xvdf /data
```

Verify

```bash
df -h
```

Expected

```
/dev/xvdf   5G   24M   /data
```

---

# Part 8 — Add Data

Create test file

```bash
sudo touch /data/testfile
```

Add data

```bash
echo "AWS EBS Lab Practice" | sudo tee /data/lab.txt
```

Verify

```bash
ls /data
```

---

# Part 9 — Permanent Mount (fstab)

Get UUID

```bash
sudo blkid /dev/xvdf
```

Example

```
UUID=ab23cd34-xxxx
```

Edit fstab

```bash
sudo vi /etc/fstab
```

Add line

```
UUID=xxxx /data ext4 defaults,nofail 0 2
```

---

# Part 10 — Validate fstab

Unmount first

```bash
sudo umount /data
```

Mount using fstab

```bash
sudo mount -a
```

Check

```bash
df -h
```

---

# Part 11 — Reverse Process (Cleanup)

## Step 1 Remove fstab entry

Edit file

```bash
sudo vi /etc/fstab
```

Delete `/data` line.

---

## Step 2 Unmount Volume

```bash
sudo umount /data
```

Verify

```bash
df -h
```

---

## Step 3 Delete Data Directory

```bash
sudo rm -rf /data
```

---

# Part 12 — Detach EBS Volume

Go to **EC2 → Volumes**

Select

```
studentname-ec2-data-volume
```

Actions → **Detach Volume**

---

# Part 13 — Delete Volume

After detach

```
Delete Volume
```

---

# Part 14 — Terminate EC2

Go to **Instances**

Select

```
studentname-ec2
```

Click

```
Terminate Instance
```

Root volume will also be deleted (if delete-on-termination enabled).
