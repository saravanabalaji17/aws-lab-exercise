Resources naming format: **studentname-***

Example (replace *studentname* with your name):

* EC2: `studentname-ec2`
* VPC: `studentname-vpc`
* Subnet: `studentname-subnet`
* NACL: `studentname-nacl`

Network flow:

```
Internet
   |
   |  (Security Group)
   |
EC2 Instance
   |
   |  (Subnet)
   |

```

Important concept:

| Feature    | Security Group | NACL                           |
| ---------- | -------------- | ------------------------------ |
| Type       | Stateful       | Stateless                      |
| Rules      | Allow only     | Allow + Deny                   |
| Evaluation | All rules      | Rule number order              |
| Priority   | No priority    | Lower number = higher priority |

# 1пёЏвѓЈ4пёЏвѓЈ Key Learning

Important NACL concepts:

1пёЏвѓЈ **Lower rule number = higher priority**

Example

```
Rule 99 в†’ evaluated first
Rule 100 в†’ evaluated second
```

---

2пёЏвѓЈ **NACL is stateless**

If inbound allows:

```
SSH 22
```

Outbound must allow:

```
Ephemeral ports (1024-65535)
```

---

3пёЏвѓЈ **Implicit DENY**

Last rule always:

```
DENY ALL
```
